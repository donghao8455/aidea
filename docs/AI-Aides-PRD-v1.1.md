## 1. 产品概述

### 1.1 产品背景

随着AI技术的火爆发展，各种新名词新概念层出不穷，如MCP、RAG、Agent等。对于程序员和大众学习者而言，这些概念难以快速理解和系统掌握。当前缺乏一个系统化、可视化的平台来展示这些概念之间的关系。

**市场痛点验证**:
- 知乎/微博上"AI新概念解释"类问题月均搜索量增长300%+
- 技术社区(V2EX/掘金)频繁出现"求科普：XXX是什么"帖子
- 现有资源分散：Wikipedia偏学术、教程视频太长、碎片化文章不系统

### 1.2 产品定位

面向大众的**零门槛AI概念科普平台**，以**可视化知识图谱**为核心，帮助用户在5分钟内理解任意AI新概念及其与其他概念的关系，降低学习门槛。

**一句话描述**: "AI领域的维基百科 + 可视化思维导图"

### 1.3 核心价值

- **降低AI学习门槛**: 让非专业用户也能系统化了解AI新名词新概念，无需技术背景
- **可视化关系展示**: 通过交互式图谱直观展示概念间的依赖、组合、演进关系
- **微学习体验**: 每个概念3-5分钟可掌握，适配碎片化时间
- **实践导向**: 每个概念配有应用场景和类比，便于理解和记忆

### 1.4 目标用户

#### 主要用户 (Primary Users)

| 用户群体 | 占比预估 | 核心特征 | 使用场景 |
|---------|---------|---------|---------|
| AI好奇的大众 | 40% | 非技术背景，25-40岁，想了解AI但不求甚深 | 通勤刷手机、社交聊天时被问及AI话题 |
| 学生群体 | 30% | 高校学生（文理科均有），需要AI基础知识 | 课程预习/复习、论文写作素材、面试准备 |
| 转行人士 | 20% | 传统行业转AI方向，有基础但需系统性梳理 | 系统学习、构建知识框架 |

#### 次要用户 (Secondary Users)

| 用户群体 | 占比预估 | 使用场景 |
|---------|---------|---------|
| 初级程序员 | 10% | 快速查阅术语含义，理解架构选型依据 |

**非目标用户** (明确排除):
- ❌ AI研究专家（内容对他们来说太浅）
- ❌ 寻找代码实现细节的开发者（我们不提供代码库）
- ❌ 寻找商业AI解决方案的企业决策者（我们不做产品推荐）

#### 关键成功因素 (Critical Success Factors)

1. **内容质量**: 每个概念的"类比"能让小白用户秒懂（自评测试通过率≥80%）
2. **图谱可用性**: 用户能在30秒内找到目标概念并理解其位置
3. **加载速度**: 移动端4G网络下首屏≤3秒
4. **SEO流量**: 6个月后自然搜索流量占比≥40%

#### 如何判断"降低AI学习门槛"是否达成？

**衡量方法**:
- **定量**: 用户在概念详情页停留时长 > 2分钟（说明有深度阅读）
- **定性**: 季度用户调研问题："使用本产品后，你对AI概念的理解程度提升了多少？"
  - 选项：①完全没帮助 ②帮助不大 ③有一些帮助 ④帮助很大 ⑤非常显著
  - 成功标准：选择④+⑤的比例 ≥ 70%

| **关系展示** | 超链接，弱关联 | 课程目录，线性 | 无 | **显式关系连线，一目了然** |
| **门槛** | 高（需要背景知识） | 中（需要时间投入） | 中（质量参差不齐） | **极低（零基础友好）** |
| **更新速度** | 慢（审核严格） | 慢（课程制作周期） | 快但杂乱 | **快且经过整理** |
| **独特卖点** | 权威性 | 系统性认证 | 时效性 | **可视化 + 零门槛 + 微学习** |

**我们的护城河**:
1. **先发优势**: 国内首个AI概念可视化图谱平台
2. **内容壁垒**: 精心编写的"类比"和"应用场景"难以被机器自动生成替代
3. **网络效应**: 概念关系图越完整，用户粘性越高
4. **SEO积累**: 长尾关键词布局，建立搜索流量护城河

#### 布局算法规格

**初始状态** (页面首次加载):
- **布局类型**: 层级树形布局（Hierarchical Layout）
- **根节点**: LLM（大语言模型）位于画布中心偏上位置
- **方向**: 从上到下（Top-to-Bottom）
- **层级分布**:
  ```
  Level 0: LLM (根节点)
  Level 1: Prompt Engineering, Chain of Thought, RAG, Agent, Fine-tuning, AI Gateway, MCP, Temperature
  Level 2: Prompt, Embedding, Vector DB, Tool Calling, Multi-Agent
  ```

**节点间距规范**:
- 同层级相邻节点最小水平间距: **100px**
- 相邻层级垂直间距: **120px**
- 自动调整算法防止节点重叠（Force-Directed微调）

**初始加载动画** (提升用户体验):
- 节点从根节点(LLM)向外逐层展开
- 每层间隔 **100ms** staggered delay
- 连线在两端节点都出现后才绘制（draw-in动画）
- 整体动画时长: **1.5秒**
- **性能约束**: 动画期间不影响用户交互（可随时中断）

interface Concept {

  };

    useCases: Array<{            // 应用场景
      title: string;             // 场景名称
      description: string;       // 1-2句说明
      example?: string;          // 真实案例 (可选)
    }>;
    
    relatedConcepts: Array<{     // 相关概念
      conceptId: string;
      relationType: 'prerequisite' | 'related' | 'component' | 'alternative';
      relationLabel: string;
    }>;
    
    resources: Array<{           // 延伸阅读
      title: string;
      url: string;
      type: 'article' | 'video' | 'paper' | 'documentation';
      language: 'zh' | 'en';
      difficulty: 'beginner' | 'intermediate' | 'advanced';
      recommended: boolean;
    }>;
  };
  
  metadata: {
    version: number;
    createdAt: string;
    updatedAt: string;
    author: string;
    reviewer: string;
    status: 'draft' | 'review' | 'published';
    wordCount: number;
  };
}

| **CI/CD** | GitHub Actions | - | 自动构建、测试、部署流水线 |

### 5.2 数据层

| 类型 | 选型 | 格式 | 说明 |
|------|------|------|------|
| **概念数据** | JSON文件 | 见4.3节Schema | 存储在 `/data/concepts/` 目录 |
| **关系数据** | JSON文件 | 见4.3节Schema | 存储在 `/data/relations.json` |
| **图数据库** | ❌ 不使用 | - | MVP阶段JSON足够，v2.0考虑引入 |

### 5.3 部署架构

```
用户浏览器 (HTTPS)
    ↓
Cloudflare CDN (可选, 静态资源缓存加速)
    ↓
Nginx (Docker Container)
    ├─ 静态资源服务 (gzip/brotli压缩)
    ├─ HTTP/2 支持
    ├─ 缓存控制头 (Cache-Control)
    └─ 安全头 (CSP, X-Frame-Options等)
    ↓
Docusaurus 构建产物 (HTML + CSS + JS + JSON + Images)
```

### 5.4 自动部署流程

GitHub Push → GitHub Actions → Docker Build → Docker Push → 服务器 Pull → Docker Run (含健康检查)

### 5.5 Docker配置要点

- 多阶段构建减小镜像体积 (目标 < 200MB)
- Nginx配置静态资源缓存 (JS/CSS 1年, HTML 1小时)
- HEALTHCHECK 健康检查机制
- 非root用户运行，只读文件系统

## 9. 错误处理机制 ✨ *新增章节*

### 错误分类和处理策略

| 错误码 | 场景 | 严重度 | 用户提示 | 处理方式 |
|--------|------|--------|---------|---------|
| E-001 | 概念详情404 | Medium | 友好404页面 + 推荐概念列表 | 返回200(SPA), meta robots:noindex |

| E-003 | 单个概念JSON缺失 | High | 该节点显示灰色占位符 | 其他节点正常, 不阻塞整体 |
| E-004 | 图片加载失败 | Low | 显示占位符图片 (灰色方块+图标) | alt文本兜底 |
| E-005 | 网络断开/离线 | Medium | 顶部Toast提示 "网络连接已断开" | 如有SW缓存则显示缓存版 |
| E-006 | 服务端5xx错误 | **Critical** | "服务器繁忙, 请稍后再试" | 每30s自动重连 (最多3次) |
| E-007 | 搜索无结果 | Info | "未找到结果" + 推荐5个热门概念 | 正常业务逻辑 |
| E-008 | JS运行时错误 | **High** | "页面出现异常, 请刷新" | React Error Boundary捕获 |
